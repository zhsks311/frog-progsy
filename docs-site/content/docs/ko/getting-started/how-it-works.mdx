FrogProgsy는 Claude Code와 AI 서비스 사이에 놓이는 **내 컴퓨터 안의 작은 연결 담당자**입니다.
Claude Code를 고쳐 쓰는 제품이 아니라, Claude Code가 원래 보내는 요청을 로컬에서 받은 뒤
대시보드에서 고른 AI 서비스에 맞게 정리해서 보내고, 돌아온 답을 다시 Claude Code가 읽을 수 있는
형태로 돌려줍니다.

```txt
Claude Code
  │
  │  "이 질문을 이 모델로 처리해 줘"
  ▼
FrogProgsy(로컬)
  │
  ├─ 어떤 AI 서비스로 보낼지 고름
  ├─ 서비스마다 다른 요청 모양을 맞춤
  ├─ 필요할 때만 웹 검색/이미지 보조 처리
  └─ 답변을 Claude Code 스트리밍 형식으로 돌려줌
  ▲
  │
선택한 AI 서비스
```

## 사용자가 실제로 겪는 흐름

1. **FrogProgsy를 켭니다.**
   `frogp start`가 로컬 연결을 열고 Claude Code가 볼 수 있는 모델 목록을 맞춥니다.

2. **대시보드에서 AI 서비스를 연결합니다.**
   OpenAI 호환 API, Anthropic, xAI, Kimi, 로컬 서버처럼 쓸 서비스를 추가하고 기본값을 고릅니다.

3. **Claude Code는 평소처럼 씁니다.**
   `claude` 명령이나 Claude Code 앱에서 요청을 보내면, Claude Code는 FrogProgsy를 하나의 로컬 모델
   입구처럼 봅니다.

4. **FrogProgsy가 목적지를 고릅니다.**
   모델 이름에 서비스가 적혀 있으면 그 서비스로 보내고, 없으면 대시보드의 기본 AI 서비스/모델을 씁니다.
   자세한 우선순위는 [모델 선택 규칙](/frog-progsy/ko/guides/model-routing/)에 있습니다.

5. **서비스에 맞게 말을 바꿔 보냅니다.**
   AI 서비스마다 요청 형식, 인증 방식, 스트리밍 이벤트 이름이 다릅니다. FrogProgsy는 그 차이를
   로컬에서 맞춰서 Claude Code가 같은 방식으로 계속 동작하게 합니다.

6. **답변이 다시 Claude Code로 돌아옵니다.**
   스트리밍 텍스트, 도구 호출, 추론 요약, 오류를 Claude Code가 기대하는 형태로 되돌려 보냅니다.

## 안쪽에서 꼭 필요한 일

FrogProgsy가 내부에서 하는 일은 크게 세 가지입니다.

- **선택**: 요청된 모델 이름과 대시보드 설정을 보고 어떤 AI 서비스/모델을 쓸지 정합니다.
- **변환**: Claude Code 쪽 요청 형식과 각 AI 서비스의 요청 형식 사이를 맞춥니다.
- **보조 처리**: 선택한 모델이 웹 검색이나 이미지 이해를 직접 처리하지 못할 때, 설정된 보조 모델로 필요한
  부분만 대신 처리할 수 있습니다.

이 과정은 모두 로컬에서 시작됩니다. FrogProgsy가 별도 원격 서버로 요청을 모아 보내는 구조가 아니며,
저장되는 정보도 로컬 설정, 인증 정보, 안전 로그, 사용량 기록처럼 제품 동작에 필요한 범위로 제한됩니다.

## 왜 Claude Code를 그대로 두나?

Claude Code 내부를 직접 바꾸면 업데이트 때마다 깨질 수 있습니다. FrogProgsy는 그 대신
**경계에서만 연결**합니다. Claude Code는 익숙한 방식으로 요청하고, FrogProgsy는 바깥쪽 AI 서비스의
차이를 흡수합니다.

이 방식의 장점은 단순합니다.

- Claude Code CLI, 앱, SDK를 같은 방식으로 쓸 수 있습니다.
- AI 서비스를 바꿔도 Claude Code 사용 습관은 거의 바뀌지 않습니다.
- 문제가 생기면 FrogProgsy 설정과 로컬 연결만 확인하면 됩니다.
- 더 자세한 내부 계약은 [내부 구조](/frog-progsy/ko/reference/architecture/)에 따로 정리되어 있습니다.
